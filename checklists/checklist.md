Чек-лист для самопроверки проекта Playlist Maker
Общая проверка
Проект компилируется и запускается без ошибок.
При запуске открывается главный экран.
Навигация между экранами работает стабильно, приложение не падает.
Навигация задаётся в классе PlaylistHost.
Главный экран (спринт 3)
На главном экране есть кнопки для перехода к поиску, плейлистам и настройкам.
При нажатии на кнопки выполняются переходы на соответствующие экраны.
В манифесте корректно прописаны все Activity.
Экран поиска (спринты 4–6)
В строке поиска отображается placeholder «Поиск».
Есть иконка «Лупа» слева и иконка «Очистить» справа.
При вводе текста можно очистить строку нажатием на «х».
При нажатии на «Лупу» запускается поиск.
Пока поиск идёт, показывается индикатор загрузки.
Если поиск завершён успешно, отображается список треков.
Если произошла ошибка, показывается сообщение об ошибке.
Отображается длительность трека в формате mm:ss.
Список треков верстается с помощью LazyColumn, элементы списка кликабельны.
Экран настроек (спринт 4)
Есть три кнопки: «Поделиться приложением», «Написать разработчикам», «Пользовательское соглашение».
Нажатие «Поделиться» открывает системное окно для выбора приложения.
Нажатие «Написать разработчикам» открывает почтовый клиент с заполненными адресом, темой и телом письма.
Нажатие «Пользовательское соглашение» открывает браузер со ссылкой.
Все строки вынесены в strings.xml.
Экран поиска. Архитектура (спринты 5–6)
Архитектура разделена на Data/Domain/UI.
Эмулятор сервера (Storage) возвращает список треков по подстроке.
Реализован репозиторий (TracksRepositoryImpl), возвращающий список треков.
В Domain есть интерфейсы NetworkClient и TracksRepository.
SearchViewModel управляет состояниями Initial, Searching, Success, Fail.
Экран поиска (SearchScreen) корректно отображает эти состояния.
FloatingActionButton и BottomSheet (спринт 7)
В проекте реализован пример экрана с FloatingActionButton.
Нажатие на кнопку вызывает действие (например, Toast).
Реализован пример экрана с ModalBottomSheet.
При нажатии на кнопку появляется панель, при клике вне панели — скрывается.
Эти компоненты используются на экране плейлистов:
FAB открывает экран создания нового плейлиста.
BottomSheet применяется для выбора плейлиста при добавлении трека.
Экран «Список плейлистов» и создание нового плейлиста (спринты 8–9)
На экране плейлистов отображается список созданных плейлистов.
Каждый плейлист показывает название и количество треков.
Есть FloatingActionButton для добавления нового плейлиста.
Экран создания плейлиста позволяет ввести имя и описание, сохранить результат.
После сохранения новый плейлист появляется в списке.
При нажатии на плейлист открывается экран Playlist details.
Экран деталей трека (спринт 8)
Экран деталей трека открывается по клику на трек в поиске.
На экране отображается название, исполнитель, время и обложка.
Есть иконка «Добавить в избранное» (работает переключение).
Есть иконка «Добавить в плейлист» — открывается BottomSheet со списком плейлистов.
Экран плейлиста (спринт 9)
Экран плейлиста открывается из списка плейлистов.
Отображается название, описание и список треков в плейлисте.
Навигация «Назад» работает корректно (возврат к списку плейлистов).
Дизайн экрана соответствует макету из Figma.
Дополнительно (если делали): работает добавление обложки для плейлиста.
Работа с базой данных (спринт 10)
В проект подключена библиотека Room.
Созданы сущности TrackEntity и PlaylistEntity и настроены DAO для работы с треками и плейлистами.
Реализован класс базы данных (AppDatabase).
Репозитории обновлены для работы с Room: поиск треков, сохранение/чтение треков и плейлистов.
Хранение плейлистов и треков переведено с моков на реальную базу данных.
Реализовано сохранение истории поиска через DataStore.
Экран «Избранное» (спринт 12)
Есть отдельный экран Favorites.
На экране отображается список всех избранных треков.
Каждый элемент списка показывает обложку, название, исполнителя и время.
Если у трека нет обложки, отображается заглушка.
Долгое нажатие на трек удаляет его из «Избранного».
Наверху экрана есть иконка «Назад» для возврата.
